# Матрица Влияния Onboarding-Вопросов

## Зачем нужны вопросы
Вопросы нужны не для анкеты ради анкеты, а для трех прикладных задач:

1. Сразу собрать рабочий профиль бизнеса без ручной настройки.
2. Сгенерировать первичный набор категорий, который можно использовать в iiko с первого дня.
3. Снизить риск неверного автомаппинга и лишних автосозданий категорий.

Ключевой принцип: ядро решения детерминированное (правила и матрица), LLM подключается только как опциональная пост-обработка.

## Что именно меняют ответы
Каждый ответ влияет на:

1. `segments` профиля (тип бизнеса).
2. `tags` профиля (операционные особенности).
3. `suggested_categories` (что предложить).
4. `matched_existing` (куда сопоставить из уже существующих).
5. `categories_to_create` (что предложить создать).
6. `risk_flags` (где нужна доп.проверка человеком).

## Бизнес-правила поверх матрицы
Чтобы не “раздувать” справочник и не плодить мусорные категории:

1. Категории не создаются молча: сначала план, потом подтверждение.
2. Роль `owner/admin` подтверждает `categories_to_create`.
3. Роль `manager` может маппить только к уже существующим категориям.
4. При отсутствии выбранного сегмента ставится `risk_flag=segment_not_selected` и включается обязательный ручной review.

## Что важно для продакшена
Вопросы сами по себе не решают качество данных. Нужна связка:

1. Анкета (`answers`) -> профиль (`segments/tags/risk_flags`).
2. Профиль -> план категорий (`suggested/matched/to_create`).
3. Подтверждение пользователем -> только потом применение в интеграции.

Именно эта схема дает контролируемое масштабирование на разные типы бизнеса, а не только на одну нишу.
