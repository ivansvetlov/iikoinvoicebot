# AGENTS.md

Правила сопровождения проекта для агентов и разработчиков.

## Каноническая документация
- Все Markdown-документы хранятся в `docs/`.
- В корне оставляем только короткий `README.md`-индекс.
- Проверенные команды запуска/диагностики фиксируются в `docs/DEBUG.md`.

## Когда обновлять документацию
Если меняется поведение, конфигурация, архитектура или операционные шаги:

- Всегда обновляйте `docs/AGENT_HANDOFF.md`.
- По необходимости обновляйте:
  - `docs/README.md`
  - `docs/TODO.md`
  - `docs/ARCHITECTURE.md`
  - `docs/DEV_SETUP.md`
  - `docs/BOT_COMMAND_MATRIX.md`
  - `docs/TESTCASES.md`

## Минимум в handoff
- Какие файлы изменены.
- Что изменилось в поведении (1-3 пункта).
- Как быстро проверить результат.

## Ограничения
- Не публиковать и не коммитить секреты.
- Не коммитить локальные артефакты: `.env`, `logs/`, `data/`, `tmp/`, `.venv/`.
