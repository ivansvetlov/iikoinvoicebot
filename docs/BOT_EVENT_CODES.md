# BOT_EVENT_CODES

Единый справочник пользовательских кодов событий Telegram-бота.

Источник истины в коде:
- `app/bot/event_codes.py`

## Коды

- `4101` (`BOT_RATE_LIMIT`)  
  Причина: превышен лимит частоты отправки файлов (`MAX_FILES_PER_MINUTE`).
  Действие: подождать и отправить позже.

- `4501` (`BOT_BACKEND_UNAVAILABLE`)  
  Причина: backend недоступен или сеть нестабильна при отправке файла(ов).
  Действие: проверить backend/worker/redis, повторить отправку.

- `4201` (`BOT_NO_PENDING`)  
  Причина: пользователь нажал кнопку обработки, но pending-буфер уже пуст.
  Действие: отправить файл заново.

- `4202` (`BOT_PENDING_TIMEOUT`)  
  Зарезервировано для сценариев таймаута pending-режима (пока не используется).

## Где это выводится пользователю

В user-facing текстах Telegram-бота через helper:
- `with_event_code("...", BOT_...)`

Это гарантирует единый формат:

```text
<человеческое сообщение>
Код: 4xxx
```

## Поиск в логах

В логах/JSONL пишутся оба значения:
- `event_short`: короткий код (`4101`, `4501`, ...)
- `event_code`: внутренний код (`BOT_...`)

Это позволяет искать и по короткому коду от пользователя, и по внутреннему коду в кодовой базе.
